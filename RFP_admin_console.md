## 1. 背景・目的

### 1-1. プロダクト概要

「100億宣言支援AI」は、中小企業庁の「100億宣言」に認可された約3,135社の中小企業が、売上100億円達成に向けて伴走することを目的とした LINE Bot 主体のサービス。提供価値は次の2系統:

1. **取り組み事例の月1配信**: ユーザー企業より売上フェーズが進んだ認可企業の取り組みを、業界・経営テーマの近さでレコメンドして配信
2. **オンライン相談会型マッチング**: 配信中に「この会社の話を聞きたい」と表明されたオファーを集約し、N件以上集まったら事務局が相談会を企画。1対多の対話の場を作る

LINE側のオンボーディング機能（友だち追加 → 会社名+URL受信 → AIプロファイル生成 → 確認）は社内チームで開発済み。**残るは事務局オペレーションを支える管理画面**で、ここを別チームに開発委託したい。

### 1-2. 本RFPの目的

事務局スタッフが日次運用するための **管理コンソール（Web）** を設計・開発するチームを募集する。提案いただいた内容を比較検討し、開発を発注する。

## 3. 機能要件

### 3-1. Must（最低限ローンチに必要）

#### A. 取り組み事例（Initiatives）の管理
- 事例の **登録 / 編集 / 削除 / 公開ステータス変更**（draft / published）
- AIで宣言PDFから半自動生成された事例ドラフトのレビュー画面
  - 元PDFの埋め込み or リンク表示
  - AIが抽出した「カテゴリ」「タイトル」「概要」「詳細」「業界タグ」「対象テーマタグ」を編集可能なフォーム
- 事例カバー画像のアップロード（任意）
- 事例の検索（フリーテキスト + ステータス + カテゴリ + 認可企業）

#### B. マッチング申請（MatchingRequests）の管理
- 申請一覧（ユーザー名、希望先企業、申請日時、ステータス）
- 認可企業ごとのオファー集約状況（X社：○件のオファー、しきい値到達フラグ）
- 申請ステータス変更（受付 / 相談会割当済 / クローズ）
- ユーザーへの個別連絡用テキスト送信機能

#### C. 相談会（ConsultationEvents）の管理
- 相談会の **作成 / 編集 / 削除**（ホスト企業、日時、Zoom URL、定員、ステータス）
- 申請者への **参加打診プッシュ通知** 一括送信
- 参加表明者一覧、出欠管理
- リマインド通知の手動送信

#### D. ユーザー（Users + Profiles）閲覧
- LINE登録ユーザー一覧（state / 認可企業 / 売上フェーズ / 登録日）
- プロファイル詳細（AI生成の業界タグ、経営テーマ等を閲覧）
- ユーザー個別への手動push送信

#### E. 認可済企業マスタの閲覧
- 検索 / 都道府県・業界・売上フェーズでフィルタ
- 詳細表示（売上、宣言PDFリンク等）
- 編集は不要（読み取り専用でOK）

#### F. 認証・権限
- 事務局スタッフのログイン（推奨：**Microsoft Entra ID** SSO 連携。代替案として、メール+パスワード）
- 最低限「管理者」「閲覧のみ」の2ロール

### 3-2. Should（できれば）

- 簡易KPIダッシュボード（登録ユーザー数、月配信数、オファー件数、相談会回数）
- 操作監査ログ（誰がいつ何をしたか）
- 配信予約スケジュール画面（毎月の配信タイミング設定 + プレビュー）

### 3-3. Could（あったら嬉しい）

- 取り組み事例の AI 再生成ボタン（プロンプト調整可）
- ユーザーへの一斉アンケート送信機能
- マッチング申請のCSVエクスポート

## 4. 既存システム概要（前提情報）

### 4-1. 技術スタック（LINE Bot 側、すでに動作中）

| 項目 | 採用 |
| --- | --- |
| ランタイム | Node.js 18+ / Express |
| DB | Azure Database for MySQL Flexible Server |
| AI | Anthropic Claude API (`claude-sonnet-4-5`) |
| LINE SDK | `@line/bot-sdk` v9 |
| デプロイ先（予定）| Azure App Service (Linux / Node 18) |
| ソース管理 | GitHub（プライベートリポジトリ）|

### 4-2. 既存DBスキーマ（一部抜粋）

```sql
-- 認可済企業マスタ（中小企業庁公開リスト、3,135件取り込み済み）
ApprovedCompanies (
  id, corporate_number, company_name, company_name_normalized,
  prefecture, industry_major, industry_minor,
  employee_count, annual_sales, target_year, declaration_pdf_url, ...
)

-- LINEユーザーごとのオンボーディング状態
Users (
  line_user_id, state, approved_company_id, sales_tier, annual_sales,
  pending_company_name, pending_company_url, pending_profile_json, ...
)

-- 確定済みプロファイル（AI生成、ユーザー承認済）
Profiles (
  id, line_user_id, approved_company_id, company_name, company_url,
  sales_tier, annual_sales, profile_json, ...
)
```

スプリント2以降で **追加予定** のテーブル（提案で設計してほしい部分も含む）:

- `Initiatives` — 取り組み事例
- `MatchingRequests` — 「話を聞きたい」申請
- `ConsultationEvents` — オンライン相談会
- `EventParticipants` — 相談会の参加者
- `DeliveryLog` / `DeliveryFeedback` — 月1配信履歴・反応

### 4-3. データ規模（想定）

- 認可企業: 約3,135社（年次更新でもほぼ横ばい）
- 取り組み事例: 初年度300〜500件、最終的に3,000件程度
- LINEユーザー: 初年度500〜1,000人、3年で3,000人想定
- マッチング申請: 月100〜500件規模
- 相談会: 月10〜30回開催規模

## 6. 技術要件・推奨スタック

**こちらから縛るのは下記のみ:**

- DB: 既存の **Azure MySQL Flexible Server** を共用（または専用DB分離も可）
- インフラ: **Azure** 上にホスト（App Service / Container Apps / Static Web Apps いずれでも可）
- 認証は **Microsoft Entra ID (Azure AD)** 連携を最優先で検討（事務局がMS環境のため）

**フレームワーク・言語は提案を歓迎:**

- 例1: **Next.js + Tailwind CSS + Auth.js + Prisma**（推奨。LINE Bot側がNode.jsなので親和性高）
- 例2: **AdminJS** + Express（既存バックエンドに乗せて簡単に立ち上げ）
- 例3: **Refine** など管理画面特化フレームワーク
- ノーコード（Retool / AppSheet / Power Apps）も検討可能。ただし長期的な拡張性とコストは要評価

提案時に **「なぜそのスタックか」** を一段書いてもらえると比較しやすいです。

## 7. LINE Bot との連携方式（提案で選んでください）

| 方式 | 概要 | 推奨度 |
| --- | --- | --- |
| A. **DB直接共有** | 管理画面が同じMySQLに直接読み書き。LINE Bot 側はテーブルを共有 | 開発速度 ◎、結合度 ×、整合性管理が要 |
| B. **API分離** | LINE Bot 側に管理用 REST API を生やし、管理画面は API 経由で操作 | 結合度 ◯、開発工数やや増、運用しやすい |
| C. **共通APIゲートウェイ** | 別途バックエンドを立て、両者がAPI経由で利用 | 拡張性 ◎、初期工数 ×、MVPには重い |

**MVPフェーズの推奨は B（API分離）**。ただし提案チームのリソースと工数で A も許容します。

push送信（LINEメッセージの一斉配信や個別通知）は LINE Bot 側に委ねるか、管理画面から直接 LINE Messaging API を叩くかは提案で選んでください。

## 付録B. 用語

- **認可済企業**: 中小企業庁の「100億宣言」リストに登録された約3,135社
- **オンボーディング**: ユーザー企業がLINE上で会社情報を提供→AIプロファイル化→確定までの一連の流れ
- **取り組み事例（Initiative）**: 認可済企業が実際に行っている経営施策。配信のネタ
- **マッチング申請**: ユーザーが「この企業の話を聞きたい」と表明したリクエスト
- **相談会**: マッチング申請が一定数集まったとき、事務局が企画する1対多のオンラインイベント
- **売上フェーズ（sales_tier）**: ユーザー企業の現在の売上規模を5段階で分類した値
