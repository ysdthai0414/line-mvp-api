# 100億宣言支援AI - 開発ロードマップ

最終更新: 2026-04-26

## プロダクトの目的

100億宣言の認可済企業（約3,135社）が **実際に売上100億円を達成する** ことを支援する。
支援は2系統:

1. **取り組み事例の月1配信**: ユーザー企業より売上フェーズが進んだ認可企業の取り組みを、
   業界・経営テーマの近さでレコメンドして配信する
2. **オンライン相談会型マッチング**: 事例配信の中で「この会社の話を聞きたい」と表明された
   オファーを集約し、N件以上集まったら事務局が当該企業との **オンライン相談会** を企画。
   参加表明者を募る形で1対多の対話の場を作る

インターフェースは LINE 公式アカウントを主軸に、必要に応じて LIFF / Web 管理画面を追加する。

## 確定済みの技術選定

- **言語/フレームワーク**: Node.js 18+ / Express / @line/bot-sdk v9
- **DB**: Azure SQL Database
- **AI**: Claude API (Anthropic) `claude-sonnet-4-5`
- **デプロイ先**: Azure（具体サービスは Phase 0 で確定）
- **認可マスタ**: 中小企業庁公開「100億宣言企業一覧」xlsx

## 全体ロードマップ

### Phase 0｜オンボーディング実機到達（現スプリント）

実機で「友だち追加 → 会社名+URL送信 → AIプロファイル確認 → 確定」が回る状態にする。

- Azureデプロイ環境の確定と構築（App Service / Container Apps / Functions のいずれか）
- Dockerfile（コンテナ運用の場合）
- 環境変数のシークレット管理（Key Vault or App Settings）
- Azure SQL に DDL 適用、認可リストを取り込み
- LINE Developers Console で Webhook URL を本番URLに差し替え
- 実機でフルフローを通す
- リッチメニュー設計（マイプロファイル / 配信履歴 / 話を聞きたい一覧 / 設定）
- 同名衝突への対応（都道府県確認フロー）

### Phase 1｜コンテンツ基盤（取り組み事例DB）

配信機能の前提となる「事例の容れ物」と「初期コンテンツ生成」を作る。

- `Initiatives` テーブル（取り組み事例）
- 宣言PDFを取得 → Claude で構造化要約するバッチ
- 抽出結果は `status='draft'` で保存し、事務局レビュー後に `published` へ
- 事務局向けの簡易レビュー画面（最初は CSV エクスポート / SQL でも可）

### Phase 2｜月1配信機能

- スケジューラ: Azure Functions Timer Trigger or Logic Apps
- レコメンド: ユーザーの `sales_tier` より上 + `industry_tags`/`management_themes` 重なり度
- Flex Message Carousel で3件提示
- `DeliveryLog`（重複配信防止）と `DeliveryFeedback`（postbackで反応回収）

### Phase 3｜「話を聞きたい」集約 + オンライン相談会

3a. オファー集約
- 事例カードに「話を聞きたい」postback ボタン
- `MatchingRequests` レコード作成
- approved_company_id ごとの未消化オファー数を監視

3b. 相談会発火
- N件達成で事務局通知（メール/Slack）
- 事務局が `ConsultationEvents` を作成（日時、Zoom URL、定員）
- 申請者全員に「相談会を開催します、参加しますか？」push
- 参加表明・キャンセル・リマインド・アーカイブ配信

### Phase 4｜LIFFアプリ（マッチング体験の拡張）

LINEだけだと一覧性が辛い局面が出てきたら着手。

- LINE Login で line_user_id 引き継ぎ
- 認可企業/取り組み事例の検索・フィルタ（売上フェーズ・業界・地域・テーマ）
- 詳細ページ・申請ボタン・自分の申請履歴

### Phase 5｜事務局オペレーション

- 事例レビュー & 公開UI（Phase 1のドラフトを処理）
- マッチング申請ダッシュボード（オファー件数、相談会企画）
- 手動push送信ツール
- KPI ダッシュボード

### Phase 6｜運用改善

- 配信フィードバック → レコメンド改善
- ユーザーの自己プロファイル更新
- 相談会後アンケート

## マッチング方針（重要）

**直接面談ではなく、集約して相談会** にする理由:

- 各認可企業（特に上位フェーズ）の経営者は時間が限られている
- 1対1のオファーが乱立すると応じきれず、ユーザー側にも待ち時間が発生する
- 1対多の相談会形式なら、聞き手も学びを共有でき、話し手も労力対効果が高い

ただし将来的には「直接1on1」も選択肢として併設する余地は残す。
（高フィー or プレミアムプラン向け）
